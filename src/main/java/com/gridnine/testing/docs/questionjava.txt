    Что такое переопределение метода?

	Переопределения или overriding  - это перезаписать функционал существующего метода,
	при этом если коротко список аргументов  и возращаемое знаячение(или подтип) в переопрделяемом методе остаются такими же.
	Чаще всего используется при имплементации интерфейсов, так же при наследовании.

    Какие бывают виды классов?

    Обычый класс, может быть в том числе Абстрактным и Финальным(final class), Интерфейс, Enum(перечисление), Nested class(вложенный): Статистический вложенный класс,
    Простой внутренний класс, Локальный класс, Анонимный класс.

    Как и зачем можно использовать модификатор final?

	F. - может применяться к переменным, параметрам методов, полям и методам класса и классам. Класс не может иметь наслдников, метод не переопределяется в классах наследниках,
	нельзя изменить значение поля после инициализации, параметры методов не могут изменить своё значение внутри метода,
	локальные переменные не будут изменены после присвоения им значения.

    Какие есть варианты использования ключевого слова try?

    Данное слово используется для отметки начала блока кода, который может привести к исключительной ситуации, используется при работе с исключительными ситуациями
    и в конструкции try - with  - resources, для автоматического закрытия ресурсов.

    Какие есть стандартные реализации интерфейса List и в каких ситуациях их нужно использовать?

    Часто используемыеб ArrayList, LinkedList, также есть Vector и Stack. В среднем ArrayList, выигрывает по скорости выполенеия операций и по потребляемой памяти,
    но LinkedList лучше использовать когда нужны частые операции вставки и удаления или в случаях когда необходимо гарантированное время добавления элемента в список.

    В чём основная идея стримов из пакета java.util.stream?

    Это возможность работать со структурами данных в функциональном стиле, основная идея упростить работу, в частности операции фильтрации, сортировки и другие.

    Каким критериям должна удовлетворять «хорошая» хэш-функция?

    Хорошая хэш функция должна уменьшить вероятность, что у разных объктов будет одинаковый результат выполнения данной функции(хорошо рассеивать значения ключей).
    Она должна вычисляться быстро.

    В чём причина популярности и широкого распространения кодировки UTF-8?

    Сокращает объём памяти, занимаемый симвлами на латинице, буквы цифры кодируються одним байтом, при этом коды соответствуют кодам ASCII,
    что даёт обратную совместимость.Является наиболее универсальной.

    Сравните форматы XML и JSON. Когда какой использовать?

    JSON- формат обмена  данных основанный на JS, а XML-язык разметки. Удобочитаемость - JSON. Если проекту требуется разметка документа и информация о методанных
    - XML(поддерживает множество сложных данных).Если более организованный обмен данных JSON.
    Самым большим недостатком XML является сложный код его синтаксического анализатора.
    JSON, простой, меньше кода, меньше размер, это ускоряет обработку, обмен,но данный формат не имеет мощных функций, связанных с проверкой и схемой, как XML.
    Для веб-сервисов, которые должны возвращать много даннх и отображать их  -JSON, идеальный формат для базы данных

    Опишите что будет происходить «под капотом» после ввода адреса сайта в браузере и нажатия Enter?

   	В адресной строке содержится информация о протоколе, хост и адрес ресурса, который хотим получить на веб сервере,на основе хоста получаем IP адрес,
   	используется DNS, если адрес уже использовался, то берётся из локального кэша, по IP устанавливаем соединение TCP(идет обмен пакетами), через порт.
   	Если взаимодействуем посредством HTPPS, идет обмен сертификатами, установка доверенного соединения,
   	формируется заголовок,HTPP, с указанием метода, расположение запрашиваемого ресурса и версии протокола,
    затем получаем ответ, ошибка/успех, в случае успеха приходит заголовок с статус кодом и HTML, браузер делает парсинг HTML. Формируется отображение веб -страницы.
